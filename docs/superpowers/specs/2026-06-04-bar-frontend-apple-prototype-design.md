# 酒吧 AI Agent 系统 Apple 风格前端原型设计

## 1. 目标

建设一个无需依赖安装即可打开查看的静态前端原型，用于展示“酒吧酒水采购、仓储与客户存酒管理 AI Agent 系统”的首版视觉方向。原型面向客户演示和内部评审，重点表达系统的信息架构、经营看板、AI Agent 能力和 Apple 风格的克制视觉语言。

## 2. 视觉原则

原型参考 `DESIGN-apple.md` 的设计语言，但不照搬消费电子产品页面。转译原则如下：

- 使用黑色全局导航和浅灰磨砂二级导航形成稳定顶部结构。
- 页面主体采用白色、浅灰和近黑色全宽分区交替，减少传统后台的厚重边框。
- 交互色只使用 Action Blue `#0066cc`，用于按钮、链接、选中状态和关键动作。
- 卡片不使用重阴影，主要通过细边框、浅灰背景和留白建立层级。
- 数据呈现保持克制，优先展示关键指标、风险、建议动作和可追溯业务入口。
- 字体使用系统字体栈，接近 SF Pro Display / SF Pro Text 的观感。

## 3. 页面结构

### 3.1 顶部导航

顶部由两层组成：

- 全局导航：黑色背景，包含系统名称、采购、库存、存酒、供应商、AI Agent、活动建议。
- 二级导航：浅灰磨砂背景，左侧显示当前页面“经营驾驶舱”，右侧提供“生成补货建议”和“新增采购单”两个操作。

### 3.2 首屏经营看板

首屏采用浅色大留白布局，突出系统定位和关键经营指标。

内容包括：

- 主标题：“今晚库存，心里有数。”
- 副标题：说明系统覆盖采购、库存、存酒和 AI 决策。
- 五个指标：缺货酒水、积压酒水、临期存酒、价格异常、AI 建议。
- 一个酒吧酒柜风格的视觉资产，使用 CSS 构建，不依赖外部图片。

### 3.3 核心业务分区

页面下方使用全宽分区展示核心模块：

- 库存预警：白色分区，展示快缺货、库存积压和批次临期。
- AI Agent 经营建议：深色分区，展示可问问题、即时结论和建议动作。
- 客户存酒提醒：浅灰分区，展示临期客户、剩余酒量和召回建议。
- 活动建议：深色分区，展示库存清理、客户召回、高毛利搭售和新品推广。
- 供应商表现：白色分区，展示价格稳定性、交付周期和推荐供应商。

### 3.4 交互行为

静态原型保留轻量交互：

- 点击导航项切换选中状态。
- 点击 Agent 问题更新回答摘要。
- 点击活动建议切换选中方案。
- 关键按钮有按压反馈。

## 4. 文件设计

- `index.html`：页面结构和静态内容。
- `styles.css`：Apple 风格设计 token、布局、响应式样式和视觉资产。
- `app.js`：导航选中、Agent 问答切换、活动建议切换等轻量交互。

## 5. 响应式设计

- 桌面端：双列首屏、全宽分区、指标横向排列。
- 平板端：首屏视觉和指标自动压缩，模块分区改为两列或单列。
- 手机端：导航横向滚动，首屏单列，所有业务卡片单列展示，按钮保持 44px 以上触控高度。

## 6. 验收标准

- 浏览器直接打开 `index.html` 能看到完整页面。
- 页面符合 Apple 风格约束：单一蓝色交互色、黑白灰分区、无装饰渐变、无重阴影。
- 首屏明确表达酒吧库存和 AI Agent 系统定位。
- 核心模块覆盖库存、存酒、供应商、活动建议和 AI 分析。
- 移动端宽度下内容不重叠、不溢出。
